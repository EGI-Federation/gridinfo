About

The main aim of GStat is to obtain information about grid services, the grid information system itself and related metrics. The system is designed in modular way so that the software can be reused in different application scenarios.

This implementation is the result of a collaboration between Academia Sinica and the Grid Deployment Group at CERN. The main purpose of the joint project is to align GStat with direction taken by the WLCG monitoring group with respect to operational tools and in addition ensure that GStat can make a contribution to middleware certification and site validation.

For more information, please take a look at the Information System Product Team portal.

If you want to provide some feedback, either positive or constructive, please email us at project-grid-info-support@cern.ch.

High-Level System Architecture

The GStat architecture is split into four main areas; core, validation, monitoring and visualization. At the core of the system is a database where the data model defines the interface between the different areas. GStat core is responsible for extracting information from the information system and maintaining the entity cache. GStat visualization uses this information for the structure of displays and GStat validation conducts various validation tests on the information content. This mechanism has been designed so that it can be also be executed on the command line and as such can be used by site administrators and automated testing suites. GStat monitoring uses the entity cache to configure the monitoring framework which leverages the Nagios framework. The results of the validation tests and stored in the database and the Nagios database is used for the monitoring results.
